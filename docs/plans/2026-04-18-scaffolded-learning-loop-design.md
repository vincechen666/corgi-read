# 中低水平英语学习者支架式学习闭环设计

## 1. 文档信息

- 日期：2026-04-18
- 阶段：方案设计
- 产品范围：桌面优先 Web 英语 PDF 阅读器
- 目标用户：中低水平英语学习者
- 核心目标：同时提升阅读理解与口头输出，降低学习挫败感

## 2. 结论摘要

当前产品已经具备一个有价值的学习雏形：`英文 PDF 输入 -> 划词理解 -> 口头复述 -> 语音转写 -> AI 反馈 -> 侧栏沉淀`。  
但这条链路更适合中高级学习者。对于中低水平用户，当前版本最大的问题不是功能太少，而是学习支架不足，导致用户会在“读不懂”或“说不出”这两个关键节点提前退出。

本次设计的核心判断是：

- 产品不应继续沿着“更强 PDF 工具 + 更强 AI 分析”扩张
- 产品应改造成一个“带支架的读懂 - 开口 - 复习闭环”
- 第一阶段重点不是增加更多功能，而是让用户更容易完成一次低挫败、可重复的学习回合

## 3. 目标用户与核心场景

### 3.1 目标用户

首要服务对象是中低水平英语学习者，这类用户通常具备以下特点：

- 愿意接触英文原文，但一遇到长句或抽象句就容易卡住
- 阅读时不只是查词困难，更常见的是句意与结构理解困难
- 想提高口语表达，但对“直接复述一段内容”容易紧张或无从下手
- 更需要可完成感、引导感和分步骤的学习体验

### 3.2 核心使用场景

用户打开一份英文 PDF，不再被默认期待“读完再整段复述”，而是围绕一个个小学习回合前进：

1. 读懂一句或一小段
2. 通过支架获得句意、结构和表达上的帮助
3. 选择一种适合自己水平的输出方式开口
4. 获得可执行的纠偏反馈
5. 将值得复习的内容自动进入复习队列

## 4. 当前产品最强的地方

当前产品最强的机制不是翻译能力本身，而是已经把真实材料输入与主动输出连接起来：

- 输入是真实的英文 PDF，而不是被简化过的练习题
- 输出要求用户真正开口，而不是只点选答案
- AI 反馈可以在一次学习会话里形成即时闭环

这条链路具备真实学习价值，也是产品最应该保留的核心资产。

## 5. 当前最关键的问题

### 5.1 读不懂时缺少分层支架

当前主要支架是划词翻译，但中低水平用户更常见的障碍并不是单词未知，而是整句结构、语义关系和重点信息难以快速抓住。只提供划词翻译会让用户频繁打断阅读，但仍然无法真正理解句子。

### 5.2 输出任务起点过高

当前主输出动作是“读完后直接复述”。这对中低水平用户过于陡峭，容易把“不会说”错误地暴露成“我学不会”，从而增加放弃概率。

### 5.3 AI 反馈偏结果展示，不够可执行

现有 AI 反馈包含纠错、语法说明和 native expression，但缺少“你下一次该怎么说”和“现在请立刻再试一次”的明确动作引导，用户看完结果后很容易结束会话，而不是继续练习。

### 5.4 侧栏沉淀更像仓库，而不是复习系统

收藏、录音和表达库可以保存内容，但没有优先级、没有节奏、没有“什么时候再看”和“先复习什么”的安排。学习资产因此只是被存下，而不是被激活。

### 5.5 首次成功体验过长

从上传 PDF 到真正感受到产品价值，中间要经过多个动作。对中低水平用户来说，首次会话如果没有快速获得“我真的学到了一点东西”的感受，就很难形成留存。

### 5.6 缺少难度调节

当前产品把所有 PDF 都当作同一难度的材料处理，中低水平用户缺少“先做简单一点的动作”的缓冲层，容易被真实材料直接劝退。

## 6. 产品方向选择

本次评估考虑了三种产品演进方向：

### 6.1 阅读辅导型

以理解支架为主，重点解决“看不懂”的问题。

- 优点：上手门槛最低
- 缺点：容易滑向翻译阅读器，削弱输出能力培养

### 6.2 平衡式学习闭环

每个学习单元固定为：`先读懂 -> 再做小输出 -> 立刻纠偏 -> 进入复习`

- 优点：最符合“阅读理解与口头输出并重”的目标
- 优点：最容易形成学习飞轮
- 缺点：需要更强的任务设计与信息层级控制

这是本次设计的推荐方向。

### 6.3 口语教练型

把口头复述、纠错和再说一遍做成主线，PDF 只是内容来源。

- 优点：短期成就感更强
- 缺点：对中低水平用户过早要求输出，挫败感高

## 7. 设计原则

- 阅读不是终点，读懂之后必须自然过渡到开口
- 输出不是“一次大复述”，而是从低门槛开始逐步升级
- AI 反馈必须指向下一步动作，而不是只做点评展示
- 沉淀必须可复习，不能只可保存
- 每次学习应围绕一个小回合设计，而不是笼统“读一份 PDF”
- 首次体验必须在数分钟内让用户完成一次完整闭环

## 8. 推荐产品骨架

推荐把产品核心单元从“阅读一份 PDF”改成“完成一个学习回合”。

一个学习回合由四步组成：

1. `理解输入`
   用户先围绕一句或一小段内容建立理解，而不是直接处理整页。
2. `低门槛输出`
   用户根据自身水平选择输出动作，如跟着说、看提示说、自己复述。
3. `立即纠偏`
   系统给出用户原句、推荐表达和可立即重说的动作入口。
4. `进入复习`
   值得复习的句子、表达和错误自动进入待复习队列。

这个结构的目标不是增加功能数量，而是让用户持续完成一轮又一轮的可执行练习。

## 9. 第一阶段 MVP 需求包

### 9.1 P0：必须优先完成

#### 需求 1：句级学习卡

当用户选中一句或一小段时，系统除了显示翻译外，还需显示：

- 这句话在说什么
- 句子结构拆解
- 1 个最值得学的表达

目的不是提供更多信息，而是帮助用户真正“读懂这句”。

#### 需求 2：三档输出入口

在当前阅读内容附近增加三种输出模式：

- `跟着说`
- `看提示说`
- `自己复述`

这样不同水平的用户都能找到可完成的起点，避免直接被整段复述压垮。

#### 需求 3：可执行反馈与“重说一遍”

AI 反馈结果需要从“展示结果”转向“驱动下一次尝试”。结果视图中应明确给出：

- 用户刚才的说法
- 更自然或更准确的推荐说法
- 立即重试的按钮或入口

#### 需求 4：首次 3 分钟学习任务

首次进入产品时，不应只看到空白阅读区和功能按钮，而应得到一个明确的小任务链：

1. 读懂一句
2. 说一句
3. 收到一次反馈
4. 保存一个表达

这样用户会更快理解产品价值。

### 9.2 P1：第二阶段优先项

#### 需求 5：待复习队列

右侧学习资产区应从静态分类升级为带优先级的复习池，至少区分：

- 今天新学的
- 说错过的
- 值得反复复习的表达

#### 需求 6：阅读难度提示

增加句子或段落级别的难度感知，帮助用户预期当前内容的挑战程度，降低随机挫败。

#### 需求 7：复述模板

为中低水平用户提供开口骨架，例如：

- `This paragraph is mainly about...`
- `The author argues that...`
- `The main point is...`

模板的作用是降低开口成本，而不是替代思考。

### 9.3 P2：后续可选

- 学习周报或成长面板
- 多文档主题学习路径
- 更细粒度的口音或发音分析

这些都不是当前 MVP 的阻塞项。

## 10. 信息架构调整建议

当前页面不应继续只是“PDF 阅读区 + 右侧沉淀栏”，而应转向三层心智结构：

### 10.1 当前阅读区

仍然以 PDF 为主，但阅读焦点应缩小到当前句或当前小段的理解任务。

### 10.2 当前任务区

页面应明确告诉用户现在该做什么，例如：

- 先看懂这句
- 选择一种方式说出来
- 听反馈后再试一次

这是现阶段最缺失的产品层。

### 10.3 学习资产区

右侧区域继续保留，但核心角色从“存档”改为“复习驱动”。

## 11. 用户旅程缺口

### 11.1 首次使用

缺少明确引导和快速成功体验。用户需要更短路径体验到“我学到了”和“我能说出来”。

### 11.2 连续使用

缺少可持续的小任务组织，用户完成一次功能体验后，不知道下一步做什么。

### 11.3 长期复习

缺少待复习、优先级和重温机制。学习资产无法积累成长期收益。

## 12. 当前不应投入的方向

以下能力容易看起来有价值，但对当前目标帮助有限，且会分散设计与研发资源：

- 复杂社交功能、排行榜、学习社区
- 重度 gamification
- 过早建设完整发音评分系统
- 太重的 PDF 管理与整理能力
- 泛化为无边界 AI 聊天助教
- 完整单词本平台化能力

## 13. 成功标准

第一阶段不以功能数量衡量成功，而以用户是否更容易完成学习回合衡量。关键指标可包括：

- 首次会话完成一次“理解 + 开口 + 反馈”的比例
- 使用输出动作的用户占比
- AI 反馈后再次开口的比例
- 学习资产进入复习队列后的再次访问率

## 14. 实施建议

后续实现应优先围绕一个完整而克制的学习回合展开，而不是多点铺开。建议顺序为：

1. 句级学习卡
2. 三档输出
3. 可执行反馈与重说
4. 待复习队列
5. 首次任务引导

只有当这条主线跑通后，再考虑难度提示、成长面板或更复杂的学习系统。
