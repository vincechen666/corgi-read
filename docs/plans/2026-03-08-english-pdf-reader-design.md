# 英语学习 PDF 阅读器原型描述文档

## 1. 文档信息

- 日期：2026-03-08
- 阶段：原型设计
- 产品形态：桌面优先 Web 页面
- 目标方向：沉浸式精读型

## 2. 产品定位

这是一个面向英语学习者的桌面端 PDF 精读阅读器，核心目标不是单纯把 PDF 文件展示出来，而是把输入型阅读与输出型口语练习连接成一个完整学习闭环。

用户在阅读英文 PDF 时，以英文原文为主，遇到不懂的词句时通过划词查看中文翻译；完成一段阅读后，可通过页面底部中央的麦克风按钮开始录音复述，系统在录音结束后执行语音转文字，并借助 AI 对复述文本进行逐句纠错、语法解析、Native 表达改写和内容点评。

产品的核心价值是：

- 帮助用户保持英文原文阅读的沉浸感
- 在需要时快速获得中文理解支持
- 鼓励用户用口头复述的方式输出理解结果
- 通过 AI 反馈帮助用户提升表达质量和口语组织能力

## 3. 目标用户与使用场景

### 3.1 目标用户

首版核心用户是希望同步提升英文阅读能力和口语表达能力的英语学习者。

这类用户的特点包括：

- 愿意读英文原文，而不是只看翻译
- 需要在阅读过程中获得即时理解支持
- 希望把“看懂”进一步转化为“能复述、能表达”
- 不一定是纯备考用户，更偏向综合英语能力提升

### 3.2 核心场景

用户打开一份英文 PDF，在阅读区沉浸式阅读原文。

阅读过程中：

- 遇到不懂的单词、短语或句子时，用户划词查看中文翻译
- 如果某条内容值得复习，可以手动收藏到右侧侧边栏

阅读一个自然段、一个页面或一个小节后：

- 用户点击底部中央麦克风按钮开始录音
- 用户口头复述自己对内容的理解
- 用户再次点击按钮结束录音
- 系统将录音转写为文本，并生成 AI 分析结果
- 页面中央弹出分析层，展示逐句纠错、语法解析、Native 改写和点评
- 关闭弹窗后，该条记录进入右侧“录音”列表，供后续回看

## 4. 设计原则

首版产品设计遵循以下原则：

- 阅读优先：PDF 阅读是主场景，其他功能不能喧宾夺主
- 按需辅助：中文翻译默认不常驻，只在用户主动触发时出现
- 输出闭环：阅读后的复述是核心学习动作，不是附属功能
- 沉淀可复习：收藏、录音记录和表达库都应可持续积累
- 首版克制：只覆盖最短学习闭环，不扩展到大而全的平台能力

## 5. 页面结构

页面采用“两栏主结构 + 底部中央悬浮主操作”的布局。

### 5.1 主阅读区

页面左侧到中部为 PDF 主阅读区，占据页面最大宽度。

其职责包括：

- 展示 PDF 页面内容
- 保持英文原文为阅读主视觉
- 支持基础阅读操作，例如翻页、缩放、页码跳转、目录入口
- 支持用户在 PDF 内容上划词触发翻译

设计要求：

- 默认不显示整段中文翻译
- 不在阅读正文中长期插入学习卡片或分析内容
- 保证视觉重心稳定，避免阅读链路被频繁打断

### 5.2 右侧学习侧边栏

页面右侧为固定宽度侧边栏，用于承载学习沉淀内容，默认常驻显示。

侧边栏采用标签切换结构，包含三个标签：

#### 录音

用于展示用户每次复述后的记录卡片，建议包含：

- 录音时间
- 关联页码或段落
- 复述内容摘要
- 查看分析详情入口

#### 收藏

用于保存用户从划词翻译中主动收藏的词句，建议包含：

- 原文内容
- 中文翻译
- 类型标记（单词、短语、句子）
- 来源页码
- 收藏时间

#### 表达库

用于沉淀 AI 从复述纠错中提炼出的高价值表达，建议包含：

- 用户原始说法
- 更自然的表达方式
- 中文说明
- 适用语境或例句
- 来源录音记录

### 5.3 底部中央主操作区

页面底部中央悬浮一个麦克风主按钮，作为整页最明确的输出入口。

其职责包括：

- 启动录音
- 结束录音
- 在录音过程中提供清晰状态反馈

设计要求：

- 视觉上需足够明确，便于形成使用习惯
- 不能长期遮挡阅读内容
- 录音中需有显著状态变化，例如颜色、脉冲动画或时长反馈

## 6. 核心交互流程

### 6.1 PDF 阅读

用户进入页面后，首先看到 PDF 主阅读区和右侧学习侧边栏。

默认状态下：

- 以英文原文阅读为主
- 中文内容不主动展示
- 右侧侧边栏可查看历史学习沉淀，但不主动抢占注意力

### 6.2 划词翻译

用户在 PDF 中选中词、短语或句子后，所选文本附近弹出轻量浮层。

浮层的核心信息包括：

- 中文翻译
- 基础释义或简要解释
- 收藏操作

交互特点：

- 以快查为主，减少跳转
- 关闭后不在正文中保留痕迹
- 用户主动点击收藏后，内容进入右侧“收藏”标签

### 6.3 录音复述

用户点击底部中央麦克风按钮后开始录音。

录音状态下：

- 麦克风按钮进入激活态
- 页面显示“录音中”反馈和录音时长
- 用户完成复述后再次点击按钮结束录音

该交互采用“点击开始，再点击结束”的方式，适合较完整的口头复述，而不是短按式语音输入。

### 6.4 录音转写与 AI 分析

录音结束后，系统执行以下流程：

1. 上传音频并进行语音转文字
2. 基于转写文本触发 AI 分析
3. 返回结构化学习反馈
4. 在页面中央展示弹窗结果

弹窗应展示以下模块：

- 用户复述原文
- 逐句纠正结果
- 语法问题解析
- 更自然的 Native 表达
- AI 内容点评

交互要求：

- 点击弹窗外区域可关闭
- 关闭后仅隐藏结果，不丢失数据
- 系统自动将本次复述记录沉淀到右侧“录音”列表

### 6.5 学习沉淀回看

用户可在右侧侧边栏中进行回看：

- 在“录音”中查看每条复述及其分析结果
- 在“收藏”中回看保存过的词句
- 在“表达库”中复习更自然的英语表达

这三类内容共同构成用户的持续学习资产。

## 7. 核心数据对象

首版原型围绕以下四类数据对象构建。

### 7.1 PDF 文档

字段建议包括：

- 文档 ID
- 文档名称
- 文件地址
- 当前阅读位置
- 当前页码
- 上传时间

用途：

- 承载阅读上下文
- 为收藏项和录音记录提供来源定位

### 7.2 收藏项

字段建议包括：

- 收藏 ID
- 原文内容
- 中文翻译
- 内容类型（单词、短语、句子）
- 来源文档 ID
- 来源页码
- 创建时间

用途：

- 保存用户主动沉淀的词句
- 支撑右侧“收藏”标签内容展示

### 7.3 录音复述记录

字段建议包括：

- 记录 ID
- 来源文档 ID
- 来源页码或段落定位
- 录音开始时间
- 录音结束时间
- 转写文本
- AI 分析结果摘要
- 详情数据引用

用途：

- 支撑右侧“录音”列表
- 支撑分析结果的二次查看

### 7.4 表达库条目

字段建议包括：

- 表达条目 ID
- 原始表达
- 更自然表达
- 中文说明
- 示例或适用场景
- 来源录音记录 ID
- 创建时间

用途：

- 将 AI 纠错结果转化为可复习的表达资产

## 8. AI 能力边界

首版 AI 能力聚焦在文本层面的学习反馈，不扩展到完整口语评测。

### 8.1 首版包含的 AI 能力

- 录音转文字
- 对转写文本做内容纠正
- 对转写文本做逐句语法分析
- 提供更自然的 Native 改写表达
- 对复述质量做内容点评

### 8.2 首版不包含的 AI 能力

- 发音打分
- 音素级口语诊断
- 实时同声纠错
- 自动生成整份 PDF 双语对照
- 全文级学习路径编排

这样可以把首版重心稳定在“阅读后表达质量提升”，避免系统复杂度过高。

## 9. 错误状态与异常处理

首版需要明确定义以下错误状态。

### 9.1 PDF 加载失败

表现：

- 文件无法解析
- 文件格式不支持
- 页面渲染失败

处理建议：

- 明确提示失败原因
- 提供重新上传入口
- 不影响其他页面框架

### 9.2 划词翻译失败

表现：

- 浮层无法返回翻译结果

处理建议：

- 在浮层内提示稍后重试
- 不打断继续阅读

### 9.3 录音权限未开启

表现：

- 用户首次点击麦克风无法开始录音

处理建议：

- 明确请求麦克风权限
- 提供浏览器授权指引

### 9.4 录音转写失败

表现：

- 录音成功结束，但无法生成转写文本

处理建议：

- 保留本次任务状态
- 允许用户重新转写或重新录音

### 9.5 AI 分析失败

表现：

- 已拿到转写文本，但 AI 反馈未成功生成

处理建议：

- 保留转写文本
- 提供“重新分析”入口
- 避免用户重复复述

### 9.6 网络较慢或处理中

表现：

- 录音结束后长时间等待

处理建议：

- 显示处理中状态
- 明确当前进度属于“转写中”还是“分析中”

## 10. 首版范围定义

### 10.1 首版必须包含

- PDF 上传与阅读
- 英文原文主视图
- 划词翻译浮层
- 收藏到右侧侧边栏
- 点击开始与结束录音
- 录音转文字
- AI 分析弹窗
- 录音记录沉淀到右侧列表
- 右侧三标签结构：录音、收藏、表达库

### 10.2 首版暂不包含

- 发音评分
- 全文双语对照模式
- 复杂笔记系统
- 学习计划和打卡系统
- 社交分享与多人协作
- 移动端专项交互优化
- 多文档并行工作区

## 11. 成功标准

如果首版原型达到以下目标，则可以认为方向成立：

- 用户能够顺畅上传并阅读英文 PDF
- 用户能在不打断阅读节奏的情况下使用划词翻译
- 用户能够完成一次完整的录音复述
- 系统能够输出结构清晰、可学习的 AI 分析结果
- 用户能够在右侧侧边栏回看自己的录音、收藏和表达沉淀
- 整体体验让用户感受到“读进去，再说出来，再被纠正”的学习闭环

## 12. 原型总结

该产品的首版不是一个泛化的 PDF 工具，也不是一个完整英语学习平台，而是一个围绕“英文精读 + 中文辅助 + 口头复述 + AI 纠错”的沉浸式学习阅读器。

其最重要的产品判断包括：

- 以英文阅读为核心，而非双语并排展示
- 以主动触发翻译为辅，而非默认全面翻译
- 以复述输出为关键学习动作，而非仅做被动理解
- 以侧边栏沉淀学习资产，而非把所有信息都堆在正文区

后续实现阶段可在此文档基础上继续拆分信息架构、页面原型、交互状态和技术实施计划。
